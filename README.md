# It takes more than two to tango: how nodule microbiomes shape legume performance
## Analysis Scripts and Data

This repository contains the R and Python scripts used to generate all figures in the manuscript:

> **"It takes more than two to tango: how nodule microbiomes shape legume performance"**

---

## Repository Structure

```
.
├── data/                          # Shared input data files
├── Figure_1/                      # Plant traits, Mantel test, Procrustes analysis
├── Figure_2/                      # Taxonomy barplots and community ordination
```

## Data Files

All shared data files are in the `data/` directory:

| File | Description |
|------|-------------|
| `normalized_kegg_results.tsv` | KO abundance matrix (hits/genome), samples × KOs |
| `normalized_kegg_results.xlsx` | Same as above in Excel format |
| `Guy_measurements.xlsx` | Plant phenotype measurements (% N, biomass, nodulation, NMF) |
| `Guy_measurements_old.xlsx` | Earlier version of plant measurements (used in Mantel analysis) |
| `ko_presence_absence_table.xlsx` | KO presence/absence per MAG bin |
| `combined_abundance_long_renormalized.tsv` | MAG bin abundance (renormalized), long format |
| `Boruta_KO_important_KOs.csv` | Boruta-selected KOs (output from `Boruta_KO_Analysis.R`) |
| `boruta_results_with_descriptions.csv` | Boruta results with KO descriptions |
| `Boruta_KO_heatmap_with_abundance_summary.csv` | Summary table from heatmap script (input to Figure 6) |
| `ko_gene_names.csv` | KO → gene name mapping |
| `Tier1_KOs_all_traits_combined.csv` | Tier-1 KOs with all trait associations combined |
| `bin_classification.csv` | MAG bin taxonomy and symbiont/endophyte classification |
| `List_of_KOs.csv` | Nitrogen metabolism gene annotations |
| `nitrogen_hmm_presence_absence.tsv` | HMM-based nitrogen gene presence/absence |
| `Inoculation_Percent.xlsx` | Nodulation percentage data for Figure 1A |
| `bacterial_vs_nonbacterial_summary.tsv` | Eukfinder read classification summary |
| `ko_pathway_module_cache.rds` | KEGG API cache: KO → pathway/module mappings |
| `pathway_to_kos_cache.rds` | KEGG API cache: pathway → all KO members |
| `ko_mag_covariation.tsv` | KO-MAG covariation table (MAG-KO attribution pipeline) |
| `ko_mag_sample_matrix.tsv` | KO × MAG × sample attribution matrix |
| `bin_diversity_KO_all_metrics_annotated.csv` | Bin diversity metrics with KO annotations (input to Supp Fig 5 heatmap) |

Figure 2-specific data files are in `Figure_2/data/`:

| File | Description |
|------|-------------|
| `combined_by_sample_genus.tsv` | SingleM genus-level community profiles, aggregated by sample |
| `combined_by_sample_species.tsv` | SingleM species-level community profiles, aggregated by sample |
| `profiles/` | Individual SingleM profile TSV files (one per sample) |

## Running Order

Some scripts produce outputs that are used as inputs to later scripts:

1. `Figure_5/Boruta_KO_Analysis.R` → produces `Boruta_KO_important_KOs.csv` (already provided in `data/`)
2. `Figure_6/map_KOs_to_pathways_modules.R` → produces KEGG caches (already provided in `data/`)
3. `Figure_6/pathway_companion_heatmap.R` → requires the above caches and `Boruta_KO_heatmap_with_abundance_summary.csv`

For Figure 1, the master script `make_figure.R` runs all sub-analyses internally.

## Notes

- **AnVIO databases**: `Supplementary_Figure_8/T6SS_operon_diagram.R` requires per-bin AnVIO SQLite databases (`consolidated_bins_anvio/*.db`) which are not included due to file size. Please contact the authors to obtain them, then update the `db_dir` variable in the script.
- **iTOL trees**: Figures 1 (tree) and Supplementary Figure 3 were generated via the [iTOL web server](https://itol.embl.de/) and are not reproducible from scripts.
- **Working directory**: Each script automatically sets its working directory to the script's own folder when run from RStudio or via `Rscript path/to/script.R`.
